# 项目管理系统 - 可视化仪表板

## 🎯 项目概述

这是一个实时监控所有项目状态与进展的可视化管理系统，基于GitHub Pages部署，每天自动更新。**新增同步系统监控模块**，提供可靠的每小时同步系统可视化监控。

## 🌐 访问地址

**GitHub Pages URL**: https://tengfeizhao1219.github.io/Projects-Dashboard/

**新增功能页面**:
- 项目总览: `index.html`
- 同步系统监控: `sync-monitor.html`

## 📊 功能特性

### 1. **项目总览面板**
- 实时监控所有项目状态
- 进度条可视化展示
- 优先级和团队分类
- 活动时间线记录

### 2. **同步系统监控** 🆕
- 可靠的每小时同步系统状态监控
- 执行成功率、平均时长、失败次数统计
- 执行时长趋势图表
- 状态分布可视化
- 执行历史记录查看
- 系统配置和性能指标

### 3. **数据统计面板**
- 项目总数统计
- 完成/进行中/受阻项目数量
- 同步系统关键指标
- 实时更新时间显示

### 4. **响应式设计**
- 适配桌面和移动设备
- 现代化UI设计
- 流畅的动画效果
- 深色主题保护眼睛

## 🔄 数据更新机制

### 自动更新
- **页面时间**: 每30秒更新一次显示时间
- **项目数据**: 每5分钟模拟项目进度更新
- **同步监控**: 每30秒自动刷新数据 🆕
- **时间线**: 自动记录重要事件

### 手动更新
通过修改相应的JavaScript数据文件：
- 项目数据: `scripts/main.js` 中的 `projects` 数组
- 同步数据: `scripts/sync-monitor.js` 中的 `mockData` 对象

## 📱 支持的项目

### 当前监控项目
1. **单词本学习系统** - 微信小程序 + 腾讯云
2. **飞书工资表系统** - 飞书集成，自动化工资计算
3. **AI记忆优化系统** - 减少token消耗的记忆管理
4. **智能网页总结插件** - Chrome浏览器扩展
5. **可视化汇报系统** - GitHub Pages自动推送
6. **自动化脚本集** - Python自动化脚本

### 新增监控系统 🆕
7. **可靠的每小时同步系统** - Linux Cron定时任务，100%可靠
   - 状态: 活跃运行中
   - 功能: 每小时自动同步任务进度
   - 监控: 实时状态查看和告警
   - 特性: 避免"失忆"，保持上下文连续性

### 项目状态说明
- **活跃 (active)**: 正常运行，进度良好
- **进行中 (in-progress)**: 正在开发，有明确进展
- **受阻 (blocked)**: 遇到问题，需要解决
- **已完成 (completed)**: 100%完成

## 🆕 同步系统监控功能

### 主要指标
- **成功率**: 98.5% (比上周提高 2.3%)
- **平均执行时长**: 1.2秒 (比上周减少 0.3秒)
- **本月失败次数**: 2次 (需要关注)
- **总执行次数**: 168次 (持续运行中)

### 图表功能
1. **执行时长趋势图**
   - 展示最近7/30/90天的执行时长变化
   - 折线图可视化，支持时间范围选择

2. **状态分布饼图**
   - 成功/警告/失败状态分布
   - 百分比显示，颜色区分

3. **执行历史表格**
   - 最近10次执行记录
   - 状态、时长、时间戳、日志文件
   - Git提交状态标记

### 控制功能
- **刷新数据**: 手动刷新监控数据
- **手动同步**: 立即触发同步任务
- **导出报告**: 下载监控报告文本文件
- **主题切换**: 切换亮色/暗色主题

## 🛠️ 部署说明

### 首次部署
```bash
# 克隆仓库
git clone https://github.com/tengfeizhao1219/projects-dashboard.git

# 进入目录
cd projects-dashboard

# 添加文件
git add .

# 提交更改
git commit -m "初始化项目管理系统 v2.0 - 新增同步监控"

# 推送到GitHub
git push origin main
```

### 启用GitHub Pages
1. 访问仓库设置: Settings → Pages
2. 分支选择: `main` 分支
3. 文件夹选择: `/ (root)`
4. 点击保存，等待部署完成

### 更新部署
```bash
# 修改文件后
git add .
git commit -m "更新项目数据或功能"
git push origin main
```
